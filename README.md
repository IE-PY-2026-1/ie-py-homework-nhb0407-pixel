## 2. 시나리오 (5~10줄)
최근 한정판 RPG 굿즈나 피규어 시장은 주식 시장만큼이나 가격 변동이 심합니다. 초보 수집가들은 적정 가격을 몰라 '바가지'를 쓰거나, 판매 타이밍을 놓쳐 손해를 보기도 합니다.
본 프로그램은 이러한 문제를 해결하기 위해 '데이터 기반의 스마트 수집 보조 도구'를 지향합니다. 사용자가 관심 있는 굿즈의 정보(정가, 현재 시세, 매물 상태 등)를 입력하면, 시스템은 축적된 데이터를 바탕으로 현재 시장의 과열 상태를 진단합니다.
특히, 본 프로그램의 핵심인 'AI 자동매매 시뮬레이션' 모드는 사용자가 설정한 '매수 목표가'와 '매도 목표가'를 기준으로 작동합니다. 시장 가격 데이터가 업데이트될 때마다 프로그램은 실시간으로 로직을 가동하여, 저평가된 매물이 나오면 자동으로 '가상 장바구니'에 담고(매수), 목표 수익률에 도달하면 자동으로 '판매 완료' 처리를 하여 자산 수익률을 극대화하는 과정을 시뮬레이션합니다. 이를 통해 사용자는 복잡한 계산 없이도 자신의 수집 포트폴리오를 효율적으로 관리하고, 최적의 매매 타이밍을 연습해볼 수 있습니다.

# 🚀 [버전별 개발 일지 & AI 협업 기록]
## 🟦 [1차 과제: V1.0] 시나리오 기획
    
# 개발 내용
변수 활용: goods_name (문자열), purchase_price (정수), profit_rate (실수) 등 다양한 자료형을 정의하고 활용했습니다.
형변환(Casting): 파이썬의 input()은 모든 입력을 문자로 취급하기 때문에, 계산을 위해 int()를 사용하여 숫자로 바꾸어 주는 핵심 문법을 포함했습니다.
f-string 포맷팅: f"{variable:,}"를 사용하여 금액에 콤마(,)를 넣었습니다.
##코드 
# =========================================================
# [Resell-Master V1.0] 굿즈 시세 분석 프로그램
# 작성자: (남현빈)
# 과제 목표: 변수 선언, 입출력, 산술 연산을 이용한 프로그램 구축
# =========================================================
# 1. 인트로 화면 출력
print("=========================================")
print("     Goods Market Analyzer V1.0")
print("      나의 굿즈 수집 포트폴리오")
print("=========================================")
print()
# 2. 데이터 입력 (input 사용)
# input()은 기본적으로 문자열로 받으므로, 숫자는 int()로 형변환이 필요합니다.
goods_name = input("1. 분석할 굿즈 이름: ")
purchase_price = int(input("2. 개당 구매 가격(원): "))
market_price = int(input("3. 현재 장터 시세(원): "))
quantity = int(input("4. 보유 수량(개): "))
# 3. 데이터 연산 (에너지/로직 처리)
# 변수를 활용하여 필요한 수치들을 계산합니다.
total_cost = purchase_price * quantity      # 총 투자 금액
total_value = market_price * quantity       # 현재 총 가치
net_profit = total_value - total_cost       # 예상 순수익
profit_rate = (net_profit / total_cost) * 100  # 수익률(%)
# 4. 최종 결과 리포트 출력
print()
print("-----------------------------------------")
print(f"[ 분석 결과: {goods_name} ]")
print(f"- 총 투자 금액: {total_cost:,}원")
print(f"- 현재 총 가치: {total_value:,}원")
print("-----------------------------------------")
print(f"▶ 예상 순수익: {net_profit:,}원")
print(f"▶ 현재 수익률: {profit_rate:.2f}%")
print("-----------------------------------------")
print("※ V2.0에서 조건문(if)을 이용한 등급 판정 기능이 추가될 예정입니다.")
print("=========================================")

## 🟩 [2차 과제: V2.0] 입출력 + 리스트 + 조건문

### ✨ 2차 과제 내용

이번 과제는 **굿즈 가성비 분석 및 매매 판정 시스템** 구현입니다.  
사용자로부터 굿즈 정보를 리스트에 입력받고,  
정가 대비 시세 비율을 계산하여 '혜자 / 적정 / 거품' 등급을 판정합니다.  
특히 `if-elif-else` 연속 조건문과 `and / or` 논리 연산자를 활용해  
목표 수익률 도달 여부까지 판단하는 지능형 매매 조언 로직을 구현했습니다.

- `for` 반복문으로 굿즈 3개 데이터를 input받아 리스트에 `append`
- `insert`, `sort`, `index`, `len()`, `sum()`, `max()` 메소드 4종 이상 활용
- `if-elif-else` 연속 조건문으로 가성비 등급(혜자/적정/거품/심각한 거품) 판정
- `and`, `or` 논리 연산자와 중첩 `if`문으로 목표 수익률 + 예산 복합 조건 판단
- `+=` 복합 대입 연산자로 총 지출 누적 계산

---

### 🤖 AI 파트너십 과정

**내용 1: 리스트 + for 활용 구조 설계**

- 프롬프트 요약: "굿즈 시세 분석 프로그램에서 사용자가 3개 굿즈 정보를 for문으로 입력받아 리스트에 append하는 구조를 짜줘. 리스트 메소드도 4종 이상 써야 해."
- 적용 내용: AI가 제안한 '빈 리스트 3개(이름/정가/시세)를 나란히 운용하고 for range(3)으로 동시에 append하는' 구조를 채택함. 데이터가 인덱스별로 일대일 대응되어 이후 조건문 분석에 바로 활용 가능한 깔끔한 구조를 확립함.

**내용 2: 가성비 등급 + 복합 조건 로직 설계**

- 프롬프트 요약: "현재가/정가 비율로 혜자/적정/거품을 판정하는 if-elif-else와, 목표 수익률 달성 여부를 and/or로 판단하는 중첩 if 구조를 같이 구현하는 법을 알려줘."
- 적용 내용: ratio(현재가÷정가) 기준 4단계 등급 분류 로직과, `profit_rate >= target AND remaining > 0` 조합의 복합 조건 판단을 실제 코드에 반영함. 단순 판정을 넘어 '예산 여유'까지 함께 고려하는 현실적인 매매 조언 로직을 완성함.

### 🛠️ Troubleshooting & 기술 회고

**문제 1: insert 후 인덱스 번호 어긋남**  
- 원인: `goods_names.insert(0, hot_item)`으로 관심 굿즈를 맨 앞에 넣으니 원래 굿즈들이 인덱스 1, 2, 3으로 밀려나 `goods_names[i]`와 `market_prices[i]`가 불일치함.  
- 해결: 출력 시 `goods_names[i+1]`로 오프셋을 +1 적용하여 해결. 리스트 병행 운용 시 insert 위치에 따라 인덱스가 달라짐을 학습함.

**문제 2: float 나눗셈 결과를 int로 오해**  
- 원인: `ratio = market_prices[i] / retail_prices[i]` 계산 시 정수끼리 나눠도 파이썬3에서는 자동으로 float가 됨을 몰랐음.  
- 해결: 파이썬3의 `/` 연산자는 항상 float 결과를 반환한다는 점을 확인하고, `{ratio:.2f}`로 소수점 2자리 포맷팅하여 출력함.
